## Description

扫码成功获得 `studentId` 后进入「欢迎确认页」，向学生展示其学号、所选 config 的基本信息（标题、题目总数、预计用时），并通过点击「开始答题」按钮触发测评会话启动（记录 `sessionStart` 时间戳、创建会话状态），随后导航到第一题。

若检测到该 `studentId` 在该 `configId` 下存在未完成的结果文件，则改为跳转到 Story 4.1 的断点续答提示页；若无法读取设备 SSAID 则阻塞并提示联系研究员。

## Acceptance Criteria

### AC1: 展示学号与测评信息并等待开始确认

**Scenario**
```gherkin
GIVEN 扫码成功后跳转到欢迎确认页
WHEN 页面渲染完成
THEN
  - 顶部展示『欢迎，{学号}』
  - 中部展示选定 config 的 title、题目总数、预计用时（所有题倒计时之和）
  - 底部显示『开始答题』按钮
  - 学生点击『开始答题』后记录测评开始时间戳，跳转到第一题
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-1.1 | 测评开始时间戳取点击『开始答题』的瞬时时间，格式 `yyyyMMdd-HHmmss`，用于结果文件命名 |
| BR-1.2 | 如果该学号在该 configId 下已有未完成的结果文件存在，跳转到断点续答提示页（Story 4.1），而非首题 |
| BR-1.3 | 预计用时按所有题（含 staged 模式两阶段）倒计时总和计算，向上取整到分钟 |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 无可读取的 SSAID | `SSAID_UNAVAILABLE` | 无法读取设备 ID，请联系研究员 | 阻塞，不允许进入作答；展示重试按钮 |

**UI Interaction**

| Trigger | Behavior |
|---------|----------|
| 页面进入 | 无入场动画 |
| 点击『开始答题』 | 按钮立即禁用，500ms 内完成跳转，避免重复点击 |

**Examples**
- Input: 学号 S001 + 5 题 config（总时长 5 分钟）
- Expected: 页面显示『欢迎，S001 / 认知记忆测评 v1 / 5 题 / 预计 5 分钟』

## Notes

### 关键技术说明（低复杂度补充）

1. **SSAID 读取时机**：建议在进入欢迎页时立即读取 `Settings.Secure.ANDROID_ID`；读取失败立即展示阻塞态（错误码 `SSAID_UNAVAILABLE`），不允许任何继续操作。Story 2.5 的 `ResultRepository.startSession()` 也会依赖该值，两处读取应共用同一获取入口（建议放在 `DeviceIdProvider` 之类的单例中）。

2. **预计用时计算（BR-1.3）**：
   - 同屏模式题目：`durationMs`
   - 分阶段模式题目：`stemDurationMs + optionsDurationMs`
   - 全部累加后除以 60000，向上取整为分钟展示
   - 计算应放入 `ViewModel`，不要在 Composable 中重复计算

3. **断点续答检测（BR-1.2）**：
   - 扫描 `/sdcard/WenJuanPro/results/` 下匹配 `{deviceId}_{studentId}_{configId}_*.txt` 的文件
   - 若存在且文件尚未达到 config 总题目数 → 视为未完成 → 路由到 Story 4.1 断点续答提示页
   - Story 4.1 尚未实现时，预留导航入口（占位路由或 TODO 注释指向 Story 4.1）

4. **会话状态传递**：`studentId` 来自 Story 2.1 的扫码结果；`configId` 与 `config` 对象来自 Story 1.4 的选择页。两者通过 Navigation 参数或共享 ViewModel（如 `SessionViewModel`）传递，避免在导航时序列化大型 config 对象。

5. **「开始答题」点击处理**：
   - 先记录 `sessionStart = System.currentTimeMillis()` 并格式化为 `yyyyMMdd-HHmmss`
   - 再调用 `ResultRepository.startSession()` 创建结果文件（Story 2.5 落地；本故事只需调用接口并处理异常）
   - 成功后导航到首题；失败（含 SSAID 缺失）保持页面并提示

6. **防重复点击**：按钮点击后立即将按钮 `enabled=false`；导航成功或失败后根据状态决定是否恢复；500ms 内完成导航意味着不应在主线程做耗时 IO（文件检测应在 IO Dispatcher 完成并短路缓存）。

### 入口与出口

- **入口**：扫码成功（Story 2.1 AC1）→ 导航到欢迎确认页，参数：`studentId`, `configId`
- **正常出口**：点击「开始答题」→ 首题页（Story 2.3/2.4）
- **备选出口**：检测到未完成结果 → Story 4.1 断点续答提示页
- **阻塞态**：SSAID 读取失败 → 停留在本页，展示错误 + 重试按钮

### 依赖校验

- 依赖 Story 2.1（提供 `studentId`）
- 依赖 Story 1.4（提供 `configId` 与 config 对象）
- 调用 Story 2.5 的 `ResultRepository.startSession()`（若 2.5 尚未完成，可先定义接口占位并在本故事中按契约调用，Dev 同步协调实现顺序）
