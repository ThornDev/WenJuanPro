**As a** 研究员，
**I want** status=not_answered/error/partial 的题目在续答时被整题重做而非跳过，
**so that** 学生有机会重新作答之前未完成的题目，避免数据缺失。

## Implementation Note

已完全实现：
- `ResultFileParser.parseCompletedQids()` 已更新为仅返回 `status=done` 的 qid（Story 4.2 修改）
- `status=not_answered/error` 的 qid 不在 completedQids 中 → cursor 不跳过 → 整题重做
- 追加写入策略（Story 2.5）保留旧行 + 追加新行 → 解析时按 qid 取最后一行
- `status=partial` 理论上不会出现（Story 2.5 BR-1.5），但若出现也不在 completedQids 中 → 重做

无新代码需要。添加 1 个 ResultFileParser 验证测试确认。
