**As a** 学生（被试），
**I want** 再次扫码同一学号时，系统自动检测到未完成的测评并提示我可以继续上次的答题进度，
**so that** 我不需要从头开始重新作答，节省时间并保留之前的答题记录。

## Epic Context

**Epic**: 4 - 断点续答

实现学生中途退出/App 异常崩溃后再次扫码同一学号自动续答的能力。

- `ResultRepository.findResumable(studentId, configId): ResumeCandidate?`（扩展）：扫描 results/ 目录，匹配文件名，解析已完成题目，返回续答候选

## Acceptance Criteria

### AC1: 扫码后自动检测是否存在未完成会话

**Scenario**
```gherkin
GIVEN 学生扫码学号 S001，已选定 configId cog-mem-2026q3
WHEN 进入欢迎确认页前先调用 ResultRepository.findResumable()
THEN
  - 扫描 /sdcard/WenJuanPro/results/ 下所有文件
  - 匹配文件名格式: {*}_{S001}_{cog-mem-2026q3}_{*}.txt
  - 若匹配到至少 1 份且其完成题数 < config 总题数 → 视为可续答；取最近一份（按时间戳最大）
  - 若所有匹配文件均已完成 → 视为新会话，跳过续答提示
  - 若无匹配 → 视为新会话
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-1.1 | 完成判定：文件中所有 config 题的 qid 均出现且 status ∈ {done} |
| BR-1.2 | 未完成判定：文件中至少有 1 题 qid 未出现，或某 qid 的 status ∈ {not_answered, error} |
| BR-1.3 | 可续答时：跳过欢迎页的『开始答题』按钮，跳转到续答提示页（AC2） |
| BR-1.4 | 续答时复用同一份结果文件，文件名（含原 sessionStart 时间戳）不变 |

**Error Handling**
| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 结果文件解析失败（损坏/半行） | `RESULT_FILE_CORRUPT` | 上次会话的记录文件已损坏，将按新会话开始 | 重命名损坏文件为 {原名}.corrupt.{时间戳}，开新会话 |

**UI Interaction**
| Trigger | Behavior |
|---------|----------|
| 扫码成功后 | 在欢迎确认页加载前调用 findResumable（< 1s 内完成） |

**Examples**
- **Input**: results/ 下有 abc_S001_cog-mem-2026q3_20260417-103045.txt（5 题中已完成 3 题）
  **Expected**: 返回 ResumeCandidate(file=该文件, completedQids=[Q1,Q2,Q3], cursor=3)
- **Input**: results/ 下无匹配文件
  **Expected**: 返回 null，走新会话流程

### AC2: 续答提示页让学生确认是否继续

**Scenario**
```gherkin
GIVEN findResumable 返回非 null 的 ResumeCandidate
WHEN 页面进入
THEN
  - 页面顶部显示『检测到未完成的测评』
  - 中部显示: 学号、测评名称、已完成 X 题 / 总 N 题、上次开始时间
  - 底部两个按钮: 『继续上次答题』（主按钮，Primary 蓝）+ 『放弃并重新开始』（次要按钮，灰）
  - 『继续上次答题』→ 跳转到首个未完成题目
  - 『放弃并重新开始』→ 弹出二次确认 → 确认后将旧文件重命名为 {原名}.abandoned.{时间戳}，开新会话
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-2.1 | 二次确认弹窗的『确认放弃』按钮放在右侧，符合 Material Design 习惯 |
| BR-2.2 | 放弃旧会话不删除文件，仅重命名（保留研究员追溯能力） |

**Error Handling**
| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 重命名旧文件失败 | `ABANDON_RENAME_FAILED` | 无法重命名旧记录，请联系研究员 | 阻塞，提示研究员处理 |

**UI Interaction**
| Trigger | Behavior |
|---------|----------|
| 页面进入 | 无入场动画 |
| 点击『继续上次答题』 | 立即跳转到首个未完成题目 |
| 点击『放弃并重新开始』 | 弹出二次确认弹窗 |

**Examples**
- **Input**: S001 + cog-mem-2026q3 + 已完成 3/5
  **Expected**: 续答提示页 + 两个按钮
- **Input**: 点击『继续上次答题』
  **Expected**: 跳转到 Q4 作答页

## Dev Notes

### Technical Constraints

- **文件扫描性能**：results/ 目录通常 < 100 文件；`listFiles()` + 文件名 regex 匹配即可，无需索引
- **文件名解析**：格式 `{deviceId}_{studentId}_{configId}_{yyyyMMdd-HHmmss}.txt`；用 regex `.*_${studentId}_${configId}_(.+)\\.txt` 匹配
- **结果文件解析**：逐行读取，`split('|')` 解析 qid 和 status 字段；按 qid 分组取最后一行
- **损坏文件处理**：解析异常时 `catch` → 重命名为 `.corrupt.{时间戳}` → 返回 null
- **IO 线程隔离**：所有文件操作必须在 `Dispatchers.IO` 上执行
- **导航参数**：续答时需传递 `studentId`, `configId`, `resumeFileName`, `cursor` 到 QuestionScreen
- **日志安全（PRD NFR9）**：Timber 日志仅打印文件名和 completedQids.size；不打印具体答案内容
