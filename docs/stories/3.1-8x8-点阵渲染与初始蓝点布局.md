**As a** 学生（被试），
**I want** 在进入记忆题时看到一个居中的 8x8 点阵网格，其中 10 个蓝色实心圆按 config 指定的位置清晰显示，其余 54 个位置以浅灰色空圆标识，
**so that** 我可以在闪烁动画开始前先观察蓝点分布、建立空间记忆锚点，为后续按顺序复现点击做准备（Epic 3 记忆题核心范式的视觉基础）。

## Epic Context

**Epic**: 3 - 记忆闪烁点击题

实现 8x8 点阵渲染、Compose 原生动画驱动的闪烁序列、1.0x 半径命中判定、学生顺序复现的完整记忆题作答闭环；满足 ±50ms 视觉时序抖动目标。

**Story Deliverables**（本故事交付 Epic 3 的**点阵视觉基础层**，是后续闪烁动画 Story 3.2 和点击判定 Story 3.3 的渲染底座）：

- `DotGrid`（`ui/components/DotGrid.kt`）：核心 Composable，使用**单个 Compose Canvas** 绘制 8x8 = 64 个点位；Props `dotsPositions: List<Int>` / `dotStates: List<DotState>` / `onTap: ((Int) -> Unit)?` / `modifier: Modifier`；通过 `BoxWithConstraints` 获取可用宽度，计算 `cellSize = availableWidth / 8`；蓝点直径 = `cellSize * 0.7`；灰点为 `cellSize * 0.7` 直径的 1dp 描边空圆；**本故事 `onTap` 固定为 `null`（点击禁用），由 Story 3.3 启用**
- `DotState`（`ui/components/DotState.kt` 或内嵌）：`enum class DotState { EMPTY, BLUE, FLASHING, SELECTED }`；本故事仅使用 `EMPTY` 和 `BLUE`；`FLASHING`（Story 3.2）和 `SELECTED`（Story 3.3）为占位
- `MemoryQuestionContent`（`ui/screens/question/MemoryQuestionContent.kt`）：记忆题 Composable 容器；顶部 `CountdownBar(isPaused = true, overlayText = "记忆中...")` + 24dp 间距 + 居中 `DotGrid`；**本故事仅渲染静态点阵 + 暂停态进度条；闪烁/复现阶段由 Story 3.2/3.3 扩展**
- `QuestionUiState` 扩展：新增 `Memory(dotsPositions, dotStates, phase: MemoryPhase, countdownProgress, isWarning, selectedSequence)` 子类型；`sealed interface MemoryPhase { data object Rendering : MemoryPhase; data class Flashing(val currentIndex: Int) : MemoryPhase; data object Recalling : MemoryPhase }`；本故事 phase 固定为 `Rendering`
- `QuestionFsm` 扩展：新增 `Enter(memory)` reduce 分支 → 初始化 `Memory` 状态 + `phase = Rendering`；**其余 memory 事件（FlashStart / FlashComplete / RecallTap / RecallTimeout）抛 `NotImplementedError` 占位**
- `QuestionViewModel` 扩展：`when (question)` 新增 `Question.Memory` 分支 → 初始化 64 个 `DotState`（`dotsPositions` 中的索引为 `BLUE`，其余为 `EMPTY`）→ 发射 `Memory` UI 状态
- `QuestionScreen` 扩展：`when (state.current)` 补 `Memory` 分支 → `MemoryQuestionContent`

## Acceptance Criteria

### AC1: 渲染 8x8 点阵并固定 10 蓝点

**Scenario**
```gherkin
GIVEN 当前题为 type=memory，dotsPositions=[3, 7, 12, 19, 22, 30, 37, 44, 51, 58]（示例）
WHEN 页面进入
THEN
  - 在屏幕中央渲染 8×8 = 64 个点位的网格
  - 网格在屏幕中央居中，宽度占屏幕宽 90%（避免边缘被系统手势区遮挡）
  - 10 个蓝点按 dotsPositions 在指定索引位置呈现（Primary 蓝 #1976D2 实心圆）
  - 其他 54 个位置为浅灰色空圆（#E0E0E0，1dp 描边，提示点位可见但无内容）
  - 蓝点直径 = 网格单元宽度 × 0.7
  - 顶部进度条静止，显示『记忆中...』文案（无倒计时）
  - 点阵无入场动画，直接渲染；蓝点立即可见
  - 点击屏幕无任何反应（闪烁阶段前禁用交互）
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-1.1 | 网格索引：0 表示左上，按行优先递增（row 0: 0-7, row 1: 8-15, ...）；63 表示右下 |
| BR-1.2 | 网格在不同屏幕尺寸下保持等比例缩放，不变形；单元格为正方形 |
| BR-1.3 | 顶部进度条（`CountdownBar`）与点阵之间留出至少 24dp 间距，避免视觉干扰 |
| BR-1.4 | `DotGrid` 使用**单个 Compose Canvas** 绘制全部 64 个点位；**禁止** 64 个独立 Composable 组件（避免过度重组） |
| BR-1.5 | 蓝点直径 = `cellSize * 0.7`（`cellSize = gridWidth / 8`）；灰点直径相同，但仅 1dp 描边无填充 |
| BR-1.6 | 蓝点颜色：Primary 蓝 `#1976D2`；灰点描边颜色：`#E0E0E0`（Gray 300） |
| BR-1.7 | 网格宽度 = 屏幕可用宽度 × 0.9（`Modifier.fillMaxWidth(0.9f)`）；网格高度 = 网格宽度（正方形） |
| BR-1.8 | `dotsPositions` 的校验已在 Story 1.3 ConfigParser 中完成（索引 0-63 范围内、无重复、恰好 10 个）；运行时不应出现非法索引 |
| BR-1.9 | 进度条使用 `CountdownBar(isPaused = true, overlayText = "记忆中...")`；进度值固定 1.0f（满格，静止不动） |
| BR-1.10 | `BackHandler(enabled = true) { }` 拦截物理返回键（复用 Story 2.3 的 `QuestionScreen` 已有拦截） |

**Data Validation**
| Field | Type | Required | Rules | Error Message |
|-------|------|----------|-------|---------------|

## Dev Notes

### Technical Constraints

- **单 Canvas 渲染（BR-1.4）**：64 个点位必须在同一个 `Canvas` Composable 内绘制；**禁止** 64 个独立 `Box` / `Circle` Composable（会导致 O(64) recomposition 在状态变化时性能灾难——Story 3.2 闪烁时每秒需多次重绘）
- **Canvas 重绘策略**：`dotStates: List<DotState>` 变化时整个 Canvas 重绘；由于单 Canvas 内绘制操作轻量（64 个 drawCircle），重绘耗时 << 1 帧（16ms）；**不需要** `drawWithCache` 优化
- **响应式缩放**：`BoxWithConstraints` → `maxWidth` → `cellSize = maxWidth / 8`；网格宽高相等（正方形）；在 360dp~600dp 屏幕上均可正确缩放
- **颜色常量**：建议在 `ui/theme/` 或 `DotGrid.kt` 内定义 `DotGridColors` object（`Blue = Color(0xFF1976D2)` / `Gray = Color(0xFFE0E0E0)` / `Flash = Color(0xFFFFB300)` / `Selected = Color(0xFF4CAF50)`）供后续 Story 复用
- **IO 线程隔离**：本故事为纯 UI 层，无 IO 操作
- **依赖白名单**：不新增任何依赖
- **动画 API 约束**：本故事无动画；但 Canvas 结构需为 Story 3.2 预留 `DotState.FLASHING` 绘制路径（黄色实心圆），以便 Story 3.2 仅需通过修改 `dotStates` 列表驱动重绘，无需重构 Canvas
- **日志内容过滤（PRD NFR9）**：Timber 日志仅打印 `qid / type / phase`；**严禁** `Timber.d(dotsPositions)` 输出完整蓝点位置（避免日志泄露测评内容）
- **性能目标**：Canvas 绘制 64 个点 < 8ms（1 帧内完成）；Story 3.2 闪烁期间帧率 >= 55 FPS（本故事为静态渲染，无帧率压力）
