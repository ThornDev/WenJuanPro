**As a** 研究员/学生（App 使用者），
**I want** 在未获得 `MANAGE_EXTERNAL_STORAGE` 权限时 App 自动识别并引导我前往系统设置完成授权，
**so that** 后续可以顺利读取 `/sdcard/WenJuanPro/config/` 下的题库并将答题结果写入 `/sdcard/WenJuanPro/results/`。

## Epic Context

**Epic**: 1 - 项目基础与配置加载

建立 Android 工程骨架与文件 IO 基础设施，交付测评集选择页作为首屏可见功能。完成本 Epic 后，研究员可独立验证 `/sdcard/WenJuanPro/config/` 下 TXT 配置是否正确，并对损坏文件获得明确的行号级错误提示。

**Story Deliverables**:

- 授权引导页（`PermissionScreen` + `PermissionViewModel`）
- 权限 Repository（`PermissionRepository` + `PermissionRepositoryImpl`）
- `CheckPermissionUseCase`（UseCase 层）
- NavHost 权限门（`permission` 为 startDestination，全局 `PermissionGate` 拦截）
- AndroidManifest 声明 `MANAGE_EXTERNAL_STORAGE`
- 授权状态在 `onResume` 重新检测

## Acceptance Criteria

### AC1: 未授权时引导用户跳转系统设置

**Scenario**
```gherkin
GIVEN App 首次启动，MANAGE_EXTERNAL_STORAGE 未授权
WHEN 用户进入 App 首屏
THEN
  - App 自动检测授权状态（Environment.isExternalStorageManager()）
  - 未授权时跳转到授权引导页（不能进入测评集选择页）
  - 授权引导页展示『需要全部文件访问权限以读取题库与写入结果』说明文案
  - 提供『前往系统设置授权』按钮，点击后跳转 ACTION_MANAGE_APP_ALL_FILES_ACCESS_PERMISSION
  - 授权引导页提供『已完成授权，重新检测』按钮
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-1.1 | 在 Android 11+ (API 30+) 使用 MANAGE_EXTERNAL_STORAGE；Android 10 及以下使用传统 READ/WRITE_EXTERNAL_STORAGE 兼容路径 |
| BR-1.2 | 授权检测发生在每次 onResume，确保用户从系统设置返回后立即生效 |
| BR-1.3 | 授权状态变更不需要重启 App |

**Error Handling**
| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 用户拒绝授权或返回时仍未授权 | PERMISSION_DENIED | 未获取存储权限，无法读取题库。请前往系统设置开启『所有文件访问权限』 | 保持在授权引导页，按钮文案变为『再次前往系统设置』 |
| 设备 ROM 不支持该 Intent | INTENT_UNAVAILABLE | 无法自动跳转到授权页面，请手动前往：设置 → 应用 → WenJuanPro → 权限 → 全部文件访问 | 展示纯文本指引，提供复制路径按钮 |

**UI Interaction**
| Trigger | Behavior |
|---------|----------|
| 首次启动且未授权 | 首屏直接显示授权引导页，无淡入动画 |
| 点击『前往系统设置授权』 | 跳转外部系统设置；返回 App 后 onResume 自动重新检测 |
| 授权成功后 | 自动导航到测评集选择页（Story 1.4），无需手动确认 |

**Examples**
- **Input**: 首次启动，无任何权限
  **Expected**: 授权引导页 + 跳转设置按钮
- **Input**: 授权后从系统设置返回
  **Expected**: 自动检测到已授权，导航到测评集选择页

## Dev Notes

### Technical Constraints

- **仅使用 Jetpack / AndroidX API**：权限检测使用 `android.os.Environment.isExternalStorageManager()`（API 30+）+ `ContextCompat.checkSelfPermission`（兼容路径），禁止引入第三方权限库 [→ coding-standards.md#关键全栈规则]
- **IO 仅在 Repository 层**：`PermissionViewModel` / `PermissionScreen` 不得直接调用 `Environment.*`；所有系统 API 访问走 `PermissionRepository` 接口 [→ coding-standards.md#IO-仅在-Repository]
- **Intent → Reducer 单向流**：UI 通过 `onIntent(Intent)` 调 VM；VM 以 `_uiState.update{}` 修改状态；不得反向调用 UI [→ coding-standards.md#Intent-Reducer]
- **Hilt Scope**：`PermissionRepository` 使用 `@Singleton`；`PermissionViewModel` 走默认 `@HiltViewModel` 作用域 [→ coding-standards.md#Hilt-Scope]
- **Timber 日志过滤**：仅打印授权状态切换（`granted=true/false`）与 Intent 可用性，**严禁**记录用户任何个人数据 [→ coding-standards.md#日志内容过滤, NFR9]
- **无淡入动画**：权限页首次渲染不得使用 `AnimatedVisibility` / `fadeIn`（front-end-spec §页面 1: 授权引导页 — 首次启动无淡入动画）[→ front-end-spec.md#页面-1-授权引导页]
- **字符串资源化**：所有用户可见文案（标题、说明、按钮、Fallback、错误消息）必须走 `strings.xml`；错误码（`PERMISSION_DENIED` / `INTENT_UNAVAILABLE`）仅进 Timber，不对用户展示 [→ error-handling-strategy.md#错误响应格式]
- **屏幕方向**：沿用 Story 1.1 已锁定的 `portrait`，不新增配置
- **白名单依赖**：无需新增任何三方库（Compose Material 3 + Hilt + Timber 已覆盖）[→ coding-standards.md#依赖白名单]

### Data Models

所有新增类型在单一文件/包内，无跨故事共享：

```kotlin
// feature/permission/PermissionUiState.kt
sealed interface PermissionUiState {
    data object Checking : PermissionUiState
    data class NotGranted(val intentAvailable: Boolean, val hasAttempted: Boolean = false) : PermissionUiState
    data object Granted : PermissionUiState
}

sealed interface PermissionIntent {
    data object CheckPermission : PermissionIntent
    data object OpenSettings : PermissionIntent
    data object Recheck : PermissionIntent
}

sealed interface PermissionEffect {
    data object NavigateToConfigList : PermissionEffect
    data class LaunchSettings(val intent: Intent) : PermissionEffect
    data object ShowIntentUnavailable : PermissionEffect
}
```

合同：

- `PermissionRepository.isExternalStorageManager(): Boolean` —— 纯查询，无副作用；API 30+ 代理 `Environment.isExternalStorageManager()`，≤29 代理 `ContextCompat.checkSelfPermission(READ_EXTERNAL_STORAGE) == GRANTED`
- `PermissionRepository.buildManageStorageIntent(): Intent?` —— Android 30+ 返回 `Intent(ACTION_MANAGE_APP_ALL_FILES_ACCESS_PERMISSION, Uri.parse("package:${applicationContext.packageName}"))` 且通过 `resolveActivity(packageManager)` 校验；无可解析 Activity 或 API ≤29 返回 `null`
