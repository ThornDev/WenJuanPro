# 用户界面设计目标

## 整体 UX 愿景

为研究现场的"低注意力 + 高时序敏感"场景设计：**学生侧**所有交互须三步以内完成（扫码 → 确认 → 作答），UI 视觉极简、无任何品牌干扰、无任何与作答无关的动效；**研究员侧**（启动配置阶段）须把"哪些 config 可用 / 哪些坏了 / 为什么坏"在 5 秒内说清。所有页面统一使用 Material 3 默认浅色主题，不引入自定义主题切换、不引入暗色模式（避免视觉对比度变化影响认知任务结果）。

## 核心交互范式

- **零文本输入**：MVP 全程不出现键盘输入；学号经摄像头扫码获得，所有控制项均为按钮/卡片点击。
- **单向流转**：进入测评后不提供"上一题"按钮，避免学生回看影响答题用时；仅"提交并下一题"或"超时自动下一题"两条路径。
- **倒计时即提示**：用顶部进度条单一视觉元素承载剩余时间，不再叠加弹窗或音效（避免干扰认知任务）。
- **错误即引导**：所有错误状态（无授权、config 损坏、扫码失败）必须给出"下一步该做什么"的明确指令，而非纯错误码。
- **隐藏式管理出口**：在首屏长按 App 图标区域 5 秒（待 PRD-V1 后细化）可进入诊断页，避免学生误触；MVP 阶段管理入口由实体返回键 + 长按结合实现，具体由 Architect 在技术细节中定义。

## 核心页面与视图

- **授权引导页**：首次启动展示 `MANAGE_EXTERNAL_STORAGE` 授权说明 + "前往系统设置"按钮。
- **测评集选择页**：列出 `/sdcard/WenJuanPro/config/` 下所有 config 卡片（含 configId、标题、题数、有效/损坏标签）。
- **扫码登录页**：全屏摄像头预览 + 简短引导文案"请将二维码对准取景框"。
- **欢迎确认页**：展示学号、configId、题目总数、预计用时、"开始答题"按钮。
- **作答页（同屏模式）**：顶部倒计时进度条 + 题干区 + 选项区 + 提交按钮。
- **作答页（分阶段模式·题干）**：顶部倒计时进度条 + 仅题干区（无选项、无操作权）。
- **作答页（分阶段模式·选项）**：顶部倒计时进度条 + 仅选项区 + 提交按钮（题干已隐藏）。
- **记忆题作答页**：8×8 点阵 + 顶部倒计时进度条；闪烁阶段全屏禁用点击；复现阶段显示已点击序号。
- **完成页**：简短"测评已完成，请将设备交还研究员"提示，无成绩反馈。
- **诊断/管理页**（隐藏入口）：当前授权状态、SSAID、最近 5 次会话日志、手动清空 results/ 按钮。

## 无障碍: 无

> MVP 不承诺 WCAG AA。研究场景的被试为视觉/触控正常的学生，且认知测评任务本身对色彩对比度、动效时序有严格控制需求，引入无障碍调整反而可能干扰实验数据。如有特殊群体测评需求，留待 V2 评估。

## 品牌设计

无对外品牌诉求。统一使用 Material 3 默认浅色主题（白底、Primary 蓝 `#1976D2` 用于按钮与点阵蓝点）；记忆题点阵蓝点固定为该 Primary 蓝；不引入 Logo、不引入欢迎动画、不引入研究机构标识（避免视觉锚点干扰认知任务）。字体使用 Android 系统默认字体（Roboto / 思源黑体 fallback）。

## 目标设备与平台: 仅移动端（Android 手机/平板，纵向）

- 首版仅支持纵向（portrait-only），横向通过 `screenOrientation="portrait"` 锁定。
- 屏幕尺寸适配 5.0" – 11.0"，最小宽度 360dp；7-10" 平板为推荐部署形态。
- 不适配折叠屏分屏 / 多窗口模式。
