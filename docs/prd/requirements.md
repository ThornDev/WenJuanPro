# 需求

## 功能性需求

- **FR1**: App 首次启动必须引导用户完成 `MANAGE_EXTERNAL_STORAGE` 授权；未授权时禁止进入作答流程，提供"前往系统设置授权"跳转入口与文案说明。
- **FR2**: App 启动后必须扫描固定路径 `/sdcard/WenJuanPro/config/` 下所有 `*.txt` 配置文件，进行 schema 校验，并以列表形式呈现可用测评集（含 configId、标题、题数）；校验失败的文件须显示具体行号与错误原因，且不可被选中。
- **FR3**: 进入测评前，App 必须调用后置摄像头扫描二维码，并将扫码内容直接作为学号；不提供任何账号/密码/注册流程；不允许键盘手输学号（避免数据污染）。
- **FR4**: 扫码后必须展示欢迎页，内容包含识别到的学号、所选 configId、测评标题、题目总数与预计用时；学生确认后才进入第一题。
- **FR5**: App 必须支持单选题：题干 + N 个选项（N 由 config 指定，默认渲染为每行 2 或 3 个），支持纯文字、纯图片、文字+图片混合三种选项形态。
- **FR6**: App 必须支持多选题：选项可勾选/取消勾选；config 中可为每个选项独立指定得分；正确答案集合用于评分但**不向学生展示对错**。
- **FR7**: App 必须支持记忆闪烁点击题：在 8×8 点阵中固定点亮 10 个蓝色圆点；这 10 个点以随机顺序依次闪烁一遍（MVP 默认每次闪烁时长 1000ms、闪烁间隔 500ms，硬编码）；闪烁结束后学生须按记忆顺序点击复现；点击命中区域**等于圆点视觉半径本身（1.0× 半径，不放大）**，宁可误判"点偏"也不放宽容错。
- **FR8**: 每道题必须支持两种呈现模式（由 config 在题级别指定）：
  - **同屏模式 (all_in_one)**：题干 + 选项同屏，单一倒计时覆盖整题；
  - **分阶段模式 (staged)**：先单独呈现题干（独立倒计时），倒计时结束后**自动进入**选项阶段（独立倒计时），学生在题干阶段无任何操作权。
- **FR9**: 作答页顶部必须显示当前阶段剩余时间的可视化进度条；进度条须随倒计时实时刷新，到 0 时自动触发阶段切换或题目结束。
- **FR10**: 同屏模式下选项倒计时归零仍未作答，记为"未作答"；分阶段模式下题干阶段倒计时归零自动进入选项阶段（此为正常流转，非异常），选项阶段倒计时归零仍未作答记为"未作答"。
- **FR11**: 每完成一题（或分阶段模式下每阶段结束）必须立即将该题/阶段记录追加写入结果文件 `/sdcard/WenJuanPro/results/{设备ID}_{学号}_{configID}_{时间戳}.txt`，并执行 `flush + fsync` 确保落盘。
- **FR12**: 结果文件命名遵循 `{设备ID}_{学号}_{configID}_{时间戳}.txt`：设备 ID 取 Android SSAID（`Settings.Secure.ANDROID_ID`）；configID 从 config 文件 header 中读取（缺失时回退为 config 文件名去扩展名）；时间戳取本次测评首次开始作答时刻，格式 `yyyyMMdd-HHmmss`。
- **FR13**: 结果文件内容须包含：每题的 qid、题型、呈现模式、题干阶段用时（仅分阶段模式）、选项阶段用时、学生答案、正确答案、得分、完成状态（done / not_answered / partial）。
- **FR14**: 同一学号在同一 configID 下重入时（结果文件已存在），App 必须自动进入"断点续答"模式：跳过已记录为 `done` 的题目，从首个未完成或 `partial` 的题目继续。
- **FR15**: 分阶段模式的"半完成题"（题干阶段已记录、选项阶段未完成或 `not_answered`）在续答时**整题重做**——题干阶段与选项阶段两个倒计时全部重新开始；该题原有 `partial` 记录被同 qid 的新记录覆盖。
- **FR16**: 配置加载失败、磁盘写入失败、扫码失败、授权失败等异常路径必须有明确的用户可见提示文案与可执行的恢复路径（重试 / 返回上一页 / 跳转设置）。
- **FR17**: 记忆题的点击命中判定使用圆点的物理坐标 + 视觉半径，命中算法须与设备 DPI 解耦；学生在闪烁阶段的任何点击都不被记录、不影响序列。

## 非功能性需求

- **NFR1**: 目标平台为 Android 8.0（API 26）至 Android 14（API 34）；仅手机/平板纵向，单 Activity + Compose 单一窗口。
- **NFR2**: 记忆题闪烁视觉时序抖动须 ≤ ±50ms（实测中位数）；不强制锁定屏幕刷新率，由 Compose 原生动画 API（`Animatable` / `InfiniteTransition` / `animate*AsState`）驱动。
- **NFR3**: 冷启动（点击桌面图标至看到首屏）≤ 2s（中端机型，¥800-¥1500 价位段平板基线）。
- **NFR4**: 单台设备须支持 ≥ 200 名学生累计作答而不丢失任何一份记录；连续 50 次完整流程（扫码 → 作答 → 落盘）零崩溃、零数据错乱。
- **NFR5**: 数据完整性：已作答题数 / 应作答题数 ≥ 99.5%；任一次中途退出后能 100% 恢复续答位置。
- **NFR6**: 答题过程中所有 IO 操作（写结果、读 config）必须在非主线程（IO Dispatcher）执行，不阻塞 UI 渲染。
- **NFR7**: 所有数据严格限于本地文件系统；App 不发起任何网络请求（包括崩溃上报、统计 SDK）；可在编译期通过依赖审查保证（无 OkHttp/Retrofit/Firebase/统计 SDK）。
- **NFR8**: APK 仅通过手动侧载（U 盘、微信、内部分发链接）分发，不上架 Google Play 或国内主流应用市场；因此 `MANAGE_EXTERNAL_STORAGE` 权限无审核风险。
- **NFR9**: 错误日志、诊断信息可写入本地 `/sdcard/WenJuanPro/.diag/`（可选目录），但不得包含学生答题的明文内容。
- **NFR10**: TXT 配置与结果文件必须采用 UTF-8 编码，不带 BOM；所有路径与文件名兼容主流国产 ROM（MIUI / EMUI / ColorOS / OriginOS）。
- **NFR11**: 代码遵循 Kotlin 官方风格，启用 ktlint；单元测试覆盖核心解析器与状态机；关键作答流程提供 1-2 条 Espresso UI 测试。
- **NFR12**: 结果文件追加写入须保证原子性（先写临时行 → fsync → 追加），App 强杀后不出现"半行"破坏文件。
