**Story type**: Frontend UI with hardware permission（Android Compose + MVI + Hilt + CameraX + ZXing）
**Applicable blind-spot categories**（per `blind-spot-categories.yaml` applicability.frontend_ui + RESOURCE emphasis due to CameraX 生命周期）:
- **High priority**: BOUNDARY, FLOW, ERROR, RESOURCE
- **Medium priority**: CONCURRENCY, DATA

**Testing battlegrounds**:
1. **JVM 单测** (`app/src/test/`)：`StandardTestDispatcher` + `runTest` + MockK — 主战场，覆盖 `StudentIdValidator` / `ScanViewModel` / `SessionStateHolder` 扩展 / NFR9 日志过滤。
2. **JVM 单测 + Robolectric** (`app/src/test/`)：`QrAnalyzer`（构造 fake `ImageProxy` + fake ZXing reader）、`CameraPermissionRepositoryImpl`（Robolectric Context + `ShadowPackageManager`）。
3. **Compose UI 测试** (`app/src/androidTest/`，`createComposeRule` 非 Hilt-based 用于状态/按钮断言；`createAndroidComposeRule<MainActivity>` + `HiltAndroidTest` 用于 NavHost 端到端路径)：覆盖 PermissionDenied / NoCamera / Recognized 三态 UI + ScanToWelcome 完整跳转 + 生命周期清理。

## Scenarios by AC

### AC1: 通过摄像头扫描二维码并将内容作为学号

**覆盖目标**：BR-1.1 ~ BR-1.10（10 条业务规则）、5 条 Error Code（STUDENT_ID_INVALID / CAMERA_DENIED / CAMERA_UNAVAILABLE / CAMERA_BIND_FAILED / CONFIG_MISSING）、6 条 UI Interaction、7 条 Examples、Data Validation 两个字段、NFR9 日志约束、Epic 2 身份录入主路径。

#### Core Unit — StudentIdValidator（正则边界）

| ID | Lvl | Pri | Test | Why |
|----|-----|-----|------|-----|
| 2.1-UNIT-001 | U | P0 | `validate("S001")` → `Valid` | 主路径最常见合法学号 |
| 2.1-UNIT-002 | U | P0 | `validate("")` → `Invalid`（BOUNDARY-001 Null/Empty） | 空串防御 |
| 2.1-UNIT-003 | U | P0 | `validate("A")` → `Valid`（1 字符 == 下界，BOUNDARY-002） | 正则 `{1,64}` 下界 |
| 2.1-UNIT-004 | U | P0 | `validate("A".repeat(64))` → `Valid`（上界，BOUNDARY-003） | 正则 `{1,64}` 上界 |
| 2.1-UNIT-005 | U | P0 | `validate("A".repeat(65))` → `Invalid`（刚越界，BOUNDARY-004） | 超长防御 — BR-1.1 |
| 2.1-UNIT-006 | U | P1 | `validate("S 001")` → `Invalid` | 空格非法（Example #2） |
| 2.1-UNIT-007 | U | P1 | `validate("学生001")` → `Invalid`（BOUNDARY-005 类型失配） | 中文 Unicode 非法 |
| 2.1-UNIT-008 | U | P1 | `validate("_-")` → `Valid`（`_` 与 `-` 为合法字符边界） | BR-1.1 允许字符边界 |

#### Core Unit — ScanViewModel（MVI 状态机 + Effect 通道）

| ID | Lvl | Pri | Test | Why |

| 2.1-UNIT-017 | U | P0 | `SavedStateHandle["configId"] = ""` → 构造即产出 `NavigateBack` Effect；**不**请求权限；`uiState` 不切 Preview | BR-1.5 + CONFIG_MISSING 错误码 |
| 2.1-UNIT-018 | U | P0 | `SavedStateHandle["configId"] = null` → 产出 `NavigateBack` Effect | BR-1.5 空值分支 |
| 2.1-UNIT-019 | U | P1 | `OnRetryPermission` → 产出 `RequestCameraPermission` Effect | UI Interaction：重新申请按钮 |
| 2.1-UNIT-020 | U | P1 | `OnOpenSettings` → 产出 `OpenAppSettings` Effect | UI Interaction：前往系统设置按钮 |
| 2.1-UNIT-021 | U | P1 | `OnCameraBindFailed` → `uiState is PermissionDenied(reason="CAMERA_BIND_FAILED")` | CAMERA_BIND_FAILED 错误码（复用 PermissionDenied 态） |
| 2.1-UNIT-022 | U | P1 | `transientSnackbar != null` 后发 `SnackbarShown` Intent → `uiState.transientSnackbar == null` | Snackbar 幂等清理（避免二次展示） |

#### Core Unit — QrAnalyzer（ImageAnalysis.Analyzer + ZXing 解码 + 去抖）

| ID | Lvl | Pri | Test | Why |
|----|-----|-----|------|-----|
| 2.1-UNIT-024 | U | P0 | 喂入 fake `ImageProxy`（YUV 单平面 `ByteArray`）+ stub ZXing reader 命中 `"S001"` → `onDecoded("S001")` 回调被调用 1 次 | QrAnalyzer 主路径 |
| 2.1-UNIT-025 | U | P0 | 同一 analyzer 实例连续 `analyze()` × 3 次（每次均命中） → `onDecoded` 回调仅 1 次（`AtomicBoolean fired`） | BR-1.10 Analyzer 层去抖（CONCURRENCY-002 atomicity） |
| 2.1-UNIT-026 | U | P1 | 构造 analyzer 时校验 ZXing `MultiFormatReader` 的 `hints` 只含 `BarcodeFormat.QR_CODE`（反射 / 注入 mock reader 验证 `setHints` 参数） | BR-1.8 禁止其他 barcode 格式 |
| 2.1-UNIT-027 | U | P1 | analyze() 内 `image.close()` 在 `finally` 中被调用（MockK `verify { imageProxy.close() }`，即使 ZXing 抛 `NotFoundException`） | RESOURCE-004 内存释放（防 ImageProxy 泄漏） |
