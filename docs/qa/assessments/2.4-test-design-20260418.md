# Test Design: 2.4 多选题渲染与作答（同屏 + 分阶段模式）

2026-04-18 | Turing (QA)

## Overview

- **Story**: 2.4 多选题渲染与作答
- **Test Design Level**: Standard（tier=medium + quality 9.23 + complexity_indicators=1 + 非 security-sensitive；Rule 2 of decisions-sm-test-design-level.yaml）
- **Story Type**: Full-stack（Compose UI + Domain FSM/UseCase + Feature ViewModel）→ BOUNDARY / ERROR / FLOW / CONCURRENCY / DATA / RESOURCE 均适用，权重 Frontend-UI 偏重
- **Delta 范围**：复用 Story 2.3 的 FSM / VM / Scaffold 骨架；本故事仅测试 **Multi** 分支差异（`Set<Int>` toggle + `ScoreMultiChoiceUseCase` + `OptionCard(isMulti=true)` + CSV 升序序列化）；2.3 通用路径（BackHandler、3 次写入失败 → Terminal、IoDispatcher 归属）**仅回归 Multi 侧**，不重测已验证的 Single 侧契约

**Totals**: 37 scenarios | Unit: 28 (75.7%) | Integration: 3 (8.1%) | E2E: 6 (16.2%)
**By Priority**: P0: 12 · P1: 17 · P2: 8
**Blind Spot**: 14 scenarios（BOUNDARY 5 / ERROR 3 / FLOW 3 / CONCURRENCY 1 / DATA 2）

## Risk Coverage

本故事无 `risk-profile` 产出（test_design_level=Standard；非 security-sensitive）。Story 已识别的 NFR3 性能约束与 NFR9 日志过滤已映射到上表：

| Risk | Mitigating Scenarios |
|------|---------------------|
| 多选评分契约（Story 2.5 预置 `answer="1,3,4"` 格式）| 2.4-UNIT-001~005 / 2.4-UNIT-013 / 2.4-INT-001 / 2.4-BLIND-DATA-002 |
| staged stem/options 阶段隔离 + stemMs 精度 | 2.4-UNIT-016~021 / 2.4-INT-002 / 2.4-E2E-003 / 2.4-BLIND-BOUNDARY-005 |
| 写入失败 3 次终止路径（复用 2.3 语义，仅多选场景回归）| 2.4-BLIND-ERROR-001/002 |
| toggle 对称性 + 提交按钮状态机 | 2.4-UNIT-007~009 / 2.4-UNIT-014 / 2.4-E2E-002 / 2.4-BLIND-BOUNDARY-004 |
| NFR3 性能（视觉反馈 ≤ 100ms）| `MultiChoiceAllInOneContentTest` 通过 Compose `runOnIdle` 隐式验证；不纳入独立 scenario |
| NFR9 日志内容过滤（`Timber.d(answer/score)` 禁止）| 静态代码审计由 QA review 阶段验证，不纳入自动化测试 |

## Execution Order

1. **P0 Unit（12 项）**：`ScoreMultiChoiceUseCase` 全分支 / FSM 全 reduce 分支 / VM 主提交路径 / 双模式 stemMs 语义
2. **P0/P1 Integration（3 项）**：all_in_one 主路径 + staged 主路径 + staged 超时
3. **P1 Unit（11 项）**：VM 副路径、staged stem 屏蔽、blind-spot FLOW/ERROR/DATA/CONCURRENCY
4. **P1 E2E（3 项）**：Compose 多选主路径 + 提交按钮状态机 + staged stem 静默点击
5. **P2（8 项）**：staged 动画、边界选项数、0 分所有选项、防御性断言、FLOW-003、stemDurationMs=0L

## Coverage Gaps（主动声明）

- **生命周期续算**（`onStop` 倒计时取消 / `onStart` 续算）：Story v1 明确不实现，Story 4.1 接管；本 test-design 不包含对应 scenario。
- **图片/混合选项渲染**（Coil）：不在本故事作用域；Compose UI 测试仅覆盖文本选项路径。
- **真正的文件写入**（原子追加 + fsync + UTF-8 行式）：Story 2.5 的 `ResultRepositoryImpl` 接管；本故事 E2E 仅用 `InMemoryResultRepository` 验证接口契约。
- **Terminal 专用路由**：复用 Story 2.3 暂不独立验证；`Effect.NavigateTerminal` 仅验证 Effect 发射。
- **2.3 已覆盖的通用路径**（倒计时进度采样精度 2.3-UNIT-014/015、warning 阈值、VM.onCleared 清理、IoDispatcher 归属、图片选项 painterResource）：本故事**不重测**，仅在多选出现异常时补充回归 scenario。
