| 风险 | 对应测试 |
|------|---------|
| Story 1.4 消费契约破坏（`ParseError.line/field/message` 命名/格式） | UNIT-005/007/011/012/016/017 — 断言字段命名、code 枚举、message 含 `{文件名}`/`{Qid}` 上下文 |

| 资产引用悬空 | BLIND-ERROR-001/002 |
| 文件 I/O 资源泄漏 | BLIND-RESOURCE-001/002 |

## Coverage Gaps

- **[GAP-1]** UNIT-018（multi `correct="3,1"` 是否允许降序）规范歧义，需 Dev 与 SM 商议并在 Dev Log 记录最终决策；本设计建议按"升序集合"判 Failure，符合 data-models.md `correctIndices: Set<Int>` 的语义但需明确输入规范。
- **[GAP-2]** BLIND-ERROR-004（`readBytes` I/O 异常的上层处理策略）未在 Story Dev Notes 中明确定义；建议 Dev 选择"捕获并包装为 Invalid + 新增 `CONFIG_FIELD_INVALID(line=0)`"或直接抛出；记录于 Dev Log。
