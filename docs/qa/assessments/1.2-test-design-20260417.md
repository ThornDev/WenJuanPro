| 1.2-BLIND-BOUNDARY-001 | BOUNDARY | P1 | I | `buildManageStorageIntent()`: `packageManager.resolveActivity(intent, 0)` returns null → method returns null (does not throw, does not return half-built Intent) | BOUNDARY-001 |
| 1.2-BLIND-BOUNDARY-002 | BOUNDARY | P1 | I | API version boundary: `Build.VERSION.SDK_INT == 30` uses `Environment.isExternalStorageManager()`; `SDK_INT == 29` uses `ContextCompat.checkSelfPermission` (no off-by-one) | BOUNDARY-002 / BR-1.1 |

| 1.2-BLIND-FLOW-001 | FLOW | P1 | U | Multi-step flow: `CheckPermission`→denied → `OpenSettings` → `Recheck` returns denied → `hasAttempted=true` → button label changes to "再次前往系统设置" | FLOW-003 |
| 1.2-BLIND-FLOW-002 | FLOW | P2 | U | Duplicate submission: two consecutive `OpenSettings` intents within 200 ms emit only one `LaunchSettings` effect (or both are safe — no crash) | FLOW-002 |
| 1.2-BLIND-FLOW-003 | FLOW | P2 | I | Compose UI: system back gesture on `PermissionScreen` does NOT navigate away (startDestination, no back stack entry to pop) | FLOW-005 |
