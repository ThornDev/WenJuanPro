# Test Design: 2.3 单选题渲染与作答（同屏 + 分阶段模式）

2026-04-18 | Turing (QA)

## Overview

- **Story**: 2.3 单选题渲染与作答
- **Test Design Level**: Standard（tier=complex + quality 10.0 + complexity=1 + 非 security-sensitive）
- **Story Type**: Full-stack（Compose UI + Domain FSM/UseCase + Data Repository/Device）→ 所有 Blind-Spot 类别适用

**Totals**: 38 scenarios | Unit: 28 (74%) | Integration: 4 (10%) | E2E: 6 (16%)
**By Priority**: P0: 14 · P1: 18 · P2: 6
**Blind Spot**: 12 scenarios（BOUNDARY 5 / ERROR 4 / FLOW 3 / CONCURRENCY 2 / DATA 3 / RESOURCE 2）

## Execution Order

1. **P0 Unit（14 项）**：Score UseCase / FSM 全部分支 / UseCase 失败转换 / SSAID 路径
2. **P0/P1 Integration（2 项）**：端到端 happy path × 2 模式
3. **P1 Unit（13 项）**：VM 主路径、staged 阶段切换、blind-spot FLOW/ERROR/DATA
4. **P1 E2E（3 项）**：Compose 主路径 × 2 + BackHandler
5. **P2（6 项）**：UI warning 色、边界选项数、staged 动画、RESOURCE/CONCURRENCY 非关键

## Coverage Gaps（主动声明）

- **生命周期续算**（`onStop` 期间倒计时 Flow 取消 / `onStart` 恢复按实际经过时间续算）：Story v1 明确**不实现**，Story 4.1 接管。本 test-design 不包含对应 scenario。
- **图片选项渲染**（Architect High Issue #1）：Coil 不在本故事作用域；`OptionContent.Image / Mixed` 分支使用 `painterResource(R.drawable.ic_placeholder)`；Compose UI 测试仅覆盖文本选项路径。
- **真正的文件写入**（原子追加 + fsync + UTF-8 行式）：Story 2.5 的 `ResultRepositoryImpl` 接管；本故事 E2E 仅用 `InMemoryResultRepository` 验证接口契约。
- **Terminal 专用路由**：本故事暂复用 `complete`；`Effect.NavigateTerminal` 仅验证 Effect 发射，不验证最终路由身份（留给 Story 2.5/6.x）。
