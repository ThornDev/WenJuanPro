# 项目简报: WenJuanPro

> 版本: v0.2 · 日期: 2026-04-17 · 作者: 孔明（业务分析师） · 模式: draft-first
>
> 变更说明（v0.2）：根据用户确认锁定以下关键决策——存储采用 `MANAGE_EXTERNAL_STORAGE` + 固定路径 `/sdcard/WenJuanPro/`（不上架 Google Play，仅手动侧载）；记忆题闪烁时长 1s、间隔 500ms 作为 MVP 硬编码默认值，点击容错采用外接矩形命中；结果文件命名格式调整为 `{设备ID}_{学号}_{时间戳}.txt`；放宽毫秒级精度约束，改用 Compose 原生动画 API，抖动阈值放宽至 ±50ms，不再锁 60Hz。

---

## 执行摘要

WenJuanPro 是一款面向科研/教学场景的 **Android 原生离线问卷与认知测评应用**。学生扫码（二维码 = 学生编号）即可登录并作答，题库与答题记录完全基于本地外部存储（TXT 行式格式）由研究员手动配置与回收，**零后端、零云同步、可断点续答**。

除常规单选/多选题外，产品的核心差异化在于内置 **「记忆闪烁点击题」**：在 8×8 点阵中固定点亮 10 个蓝色圆点，随后这 10 个点以随机顺序依次闪烁一遍（MVP 默认闪烁时长 1s、闪烁间隔 500ms），学生需凭视觉短时记忆按闪烁顺序复现点击。

- **产品概念**：一句话描述——"一台手机、一份 TXT、一次扫码，即可在任何无网环境采集认知行为数据。"
- **待解决的主要问题**：科研/教学现场的测评常受限于网络、服务端配置成本与数据隔离合规；现有通用问卷工具缺乏记忆类任务范式与离线落盘能力。
- **目标市场**：高校认知科学/心理学实验室、基础教育研究机构、企业 HR/培训认知评估场景。
- **核心价值主张**：**离线优先 + 可配置题型 + 阶段化倒计时 + 断点续答**，让研究员以最低运维成本获取结构化的作答与用时数据。

## 问题陈述

### 现状与痛点
- 科研现场常处于**弱/无网络环境**（教室、田野点、隔离受试空间），而主流问卷平台强依赖在线服务。
- 通用问卷工具（Wenjuanxing、SurveyMonkey 等）不支持**认知测评范式**，尤其缺少基于倒计时与顺序复现的视觉闪烁任务。
- 自研 Web 方案通常需要搭建后端、处理鉴权与同步逻辑，**对小团队研究员的开发/运维门槛高**。
- 设备间配置下发与版本差异容易造成数据污染（被试 A 与被试 B 题序/时长不一致）。

### 问题影响
- 数据丢失或不一致：研究员需手动对齐设备；一次现场失误往往意味着整批样本报废。
- 研究迭代慢：调整一题或一个时长都可能牵涉服务端发版、测试与分发。
- 合规压力：测评数据（尤其含未成年人）上云存在数据出境/留存期限的灰色地带。

### 现有方案为何不足
- **在线问卷类**：依赖网络；无毫秒级时序；无图像+倒计时的阶段化题型。
- **心理学实验软件（PsychoPy、E-Prime）**：PC 桌面为主，现场难以规模化发放平板/手机。
- **自研 App**：一次性成本高，复用性差；题目迭代需重新编译/发版。

### 为何现在做
- Android 设备（含低端平板）在教学/科研现场已普及，单位成本 ≤ ¥1,000；
- Jetpack Compose 原生动画 API 已能稳定承载闪烁/倒计时呈现，视觉时序满足研究可接受范围（百毫秒级）；
- 研究员侧对"**不用学习配题工具、直接改 TXT**"的诉求强烈，符合最小工具链原则。

## 解决方案

### 核心思路
以**文件驱动**的纯本地 Android App 承载完整作答流程：

1. **配置即文本**：题库、素材、结果全部落在 `/sdcard/WenJuanPro/` 三个子目录（`config/` `assets/` `results/`）中，研究员像写 CSV 一样直接编辑 TXT 即可发布/修改题目。设备首次启动申请 `MANAGE_EXTERNAL_STORAGE` 授权（仅侧载分发，不走 Google Play）。
2. **扫码即身份**：二维码即学生编号，无账号体系、无密码、无后台鉴权；重入同一编号即自动续答。
3. **原生动画驱动的视觉呈现**：基于 Kotlin + Jetpack Compose 原生动画 API 控制记忆题闪烁（MVP 默认闪烁时长 1s、间隔 500ms）与倒计时进度条，呈现抖动控制在 ±50ms 量级即可满足研究需求。
4. **阶段化可配置的题目呈现**：每题可独立选择"同屏模式"或"题干/选项分阶段模式"，每阶段独立倒计时；记录系统相应区分记录每阶段用时。
5. **实时写入、断点续答**：每一题作答即 flush 到结果文件（命名：`{设备ID}_{学号}_{测评时间戳}.txt`），App 异常退出或学生中途离开后，再次扫码重入自动定位到未作答题目继续。

### 与现有方案的关键差异
| 维度 | 在线问卷 | 心理实验软件 (PC) | WenJuanPro |
|---|---|---|---|
| 部署环境 | 强依赖网络 | 实验室 PC | **任何 Android 设备、完全离线** |
| 视觉时序 | 较弱 | 精确 | **Compose 动画驱动，满足研究级可接受范围** |
| 配题门槛 | GUI 编辑 | 脚本编写 | **直接写 TXT** |
| 断点续答 | 弱 | 无 | **原生支持** |
| 数据出境风险 | 高 | 低 | **无** |

### 成功的逻辑
- **最小化运维**：没有后端就没有可宕机、可被黑、可合规追责的服务边界。
- **研究员直连数据**：研究员既是配题者也是数据消费者，TXT 双向可读写，无工具链隔阂。
- **范式可扩展**：记忆闪烁点击题验证了"阶段化呈现 + 空间复现"的技术可行性，后续可扩展到其他认知范式（N-back、Stroop 等）。

### 高层愿景
成为**国内研究员在离线场景下首选的移动端认知测评壳**——研究员只需写 TXT，设备上自动呈现合规、可复现、可追溯的测评流程。

## 目标用户

### 主要用户群体: 研究员 / 配题者
- **画像**：高校心理学/认知科学/教育学实验室的硕博生与科研教师；具备基础数据分析能力（会用 Excel/R/Python），但**不一定会写代码**。
- **当前行为/流程**：常用 PsychoPy、问卷星、自研 H5；在现场用 U 盘拷配置、手动整理结果。
- **需求与痛点**：
  - 不想学新的配题 GUI，更希望**文本可编辑、可版本化**；
  - 现场希望**一次部署多台设备**，题目内容统一；
  - 对**记录完整性**（含每阶段用时）敏感。
- **目标**：在 2 周内完成一轮 N=50 的现场测评，数据可直接进入统计分析流水线。

### 次要用户群体: 学生 / 被试
- **画像**：K12 至大学阶段的学生，或任何能识别二维码并在触屏上点击的被试；单次参与时长 5–15 分钟。
- **当前行为/流程**：扫码 → 完成 → 离开；对"登录/注册"等流程敏感，易流失。
- **需求与痛点**：
  - 流程必须**极简**：扫码即开始；
  - 中途被打断（老师叫、上厕所）不希望数据作废；
  - 认知类任务需要**清晰的视觉引导**与**无干扰背景**。
- **目标**：顺畅地完成测评，不因设备或流程问题产生焦虑。

## 目标与成功指标

### 业务目标
- 在 2026 Q3 前完成 MVP 上线，支撑至少 **2 个真实科研课题**落地使用。
- 单台设备支持 ≥ 200 名学生累计作答而不丢失/错乱任何一份记录（可靠性 100%）。
- 研究员从拿到 APK 到完成第一轮测评的首次成功周期 ≤ **1 个工作日**。

### 用户成功指标
- **研究员**：配题 → 首轮数据回收，无需修改 App 即可完成；配置错误可本地自检发现（启动时校验 TXT）。
- **学生**：MVP 阶段作答流失率（开始后未完成）< 5%；扫码到开始答题 ≤ 3 秒。

### 关键绩效指标 (KPI)
- **数据完整率**: 已作答题数 / 应作答题数，目标 ≥ 99.5%；
- **闪烁时序抖动**: 记忆题每次闪烁实际点亮时长相对默认 1s 的偏差，目标 ≤ ±50ms（不锁定屏幕刷新率）；
- **断点续答命中率**: 中途退出的会话中成功续答完成的比例，目标 ≥ 95%；
- **配题到上线周期**: 研究员编辑 TXT → 设备识别新题目的时延，目标 ≤ 1 分钟（重启 App 内）。

## MVP 范围

### 核心功能（必须有）
- **扫码登录**: 使用设备摄像头扫描二维码并将内容作为学生编号，直接进入欢迎页；无账号/密码/注册流程。
- **欢迎页 & 开始答题**: 展示学号与测评信息；确认后进入作答。
- **三种题型**:
  - **单选题**: 题干 + N 个选项（N ∈ {4, 5, N}），每行 2 或 3 个，支持文字/图片/混合；
  - **多选题**: 多选版本，含每选项得分与正确答案集合；
  - **记忆闪烁点击题**: 8×8 点阵，固定点亮 10 个蓝点 → 随机顺序依次闪烁一遍（MVP 默认闪烁时长 1s、闪烁间隔 500ms，硬编码）→ 学生按记忆顺序点击复现；点击命中判定采用**圆点外接矩形（可略大于视觉半径的命中区）**即判中。
- **倒计时与呈现模式**: 每题可配置 **A) 同屏模式**（题干+选项同屏、一个倒计时）或 **B) 分阶段模式**（先题干独立倒计时 → 再选项独立倒计时）。
- **顶部倒计时进度条**: 当前阶段剩余时间可视化。
- **外部存储题库**: 首次启动申请 `MANAGE_EXTERNAL_STORAGE` 权限；从固定路径 `/sdcard/WenJuanPro/config/*.txt` 加载；素材引用 `/sdcard/WenJuanPro/assets/`；启动时做格式校验并给出友好报错。
- **答题记录实时落盘**: 每答完一题（或每阶段结束）立即写入 `/sdcard/WenJuanPro/results/{设备ID}_{学号}_{测评时间戳}.txt`；设备 ID 取 Android SSAID（ANDROID_ID），测评时间戳为本次测评首次开始答题的时间（`yyyyMMdd-HHmmss`）；字段含学生答案、正确答案、得分、作答用时（分阶段模式分别记录题干阶段与选项阶段用时）。
- **断点续答**: 同一学号（配合同一 config 的测评会话）重入时读取对应 `results/*.txt`，跳过已完成题目，从中断点继续。
- **视觉时序**: 记忆题闪烁由 Jetpack Compose 原生动画 API 驱动，抖动控制在 ±50ms 量级即可满足研究需求，不锁定屏幕刷新率。

### MVP 范围外
- 云端同步、账号体系、服务端管理后台；
- GUI 配题工具（研究员直接编辑 TXT）；
- 多语言界面（MVP 仅中文）；
- 复杂统计分析/可视化（研究员用外部工具处理 `results/*.txt`）；
- iOS / Web / 平板横屏自适应之外的多端支持；
- OTA / 自动更新（通过 APK 安装升级）；
- 被试端的实时提示/反馈（正确/错误弹窗），以免干扰测评范式。

### MVP 成功标准
- 在至少 1 款主流中低端 Android 设备（如 ¥800–¥1,500 价位段平板）上连续完成 50 次扫码 → 作答 → 落盘全流程，无崩溃、无数据丢失；
- 闪烁时序抖动在实测中 ≤ ±50ms；
- 研究员用提供的 TXT 模板能独立完成一套新题目的配置与回收；
- 任一次中途退出后能 100% 恢复续答位置。

- 面向**基础教育区域评估**的批量设备管理与模板；

## 技术考量

### 平台要求
- **目标平台**: Android 原生，最低 Android 8.0 (API 26)，目标 Android 14 (API 34)；仅手机/平板纵向（首版）。
- **浏览器/系统支持**: 不适用（原生 App）。需具备后置摄像头（扫码）与外部存储读写权限（`MANAGE_EXTERNAL_STORAGE`）。
- **性能要求**: 闪烁时序抖动 ≤ ±50ms（不锁定屏幕刷新率）；冷启动 ≤ 2s；作答页操作响应顺滑无明显卡顿即可。

### 技术偏好
- **前端**: Kotlin + Jetpack Compose（Material 3）；扫码用 CameraX + ML Kit Barcode（或 ZXing 兜底）；记忆题闪烁与倒计时进度条由 **Compose 原生动画 API**（`animate*AsState` / `Animatable` / `InfiniteTransition` 等）驱动，不使用 `Choreographer`/`withFrameMillis` 做帧级手动时序。
- **后端**: 无（纯离线）。
- **数据库**: 无关系型数据库；**单一事实源为 TXT 文件**（`config/*.txt`、`results/{设备ID}_{学号}_{时间戳}.txt`）。可选在内存使用 `kotlinx.serialization` 解析后的数据结构，不做本地 SQLite 持久层（避免与 TXT 产生"双写不一致"）。
- **托管/基础设施**: APK 通过 U 盘/微信/内部分发（**不上架 Google Play 或国内应用市场**，因此 `MANAGE_EXTERNAL_STORAGE` 无审核风险）；无服务端。

### 架构考量
- **仓库结构**: 单仓单模块起步（`app/`），后期按题型拆 feature module。
- **服务架构**: App 内部采用 MVI/单向数据流（Compose 友好）；IO 层抽象为 `QuestionRepository`/`ResultRepository`，屏蔽 TXT 解析细节。
- **集成需求**: 仅与本地文件系统集成；需申请摄像头权限 + `MANAGE_EXTERNAL_STORAGE`（首次启动引导用户到系统设置授权）；路径固定为 `/sdcard/WenJuanPro/{config,assets,results}/`。
- **安全/合规**:
  - 数据完全本地，**不出设备**；
  - 学生编号为研究员侧可控标识，建议使用匿名编号而非姓名/身份证；
  - 结果文件名含 Android SSAID，便于多设备回收后溯源；研究员应确保 SSAID 与设备的对应关系记录；
  - 设备交接时由研究员负责清理 `results/` 目录。

## 约束与假设

### 约束条件
- **预算**: 小型研究团队自有预算；不支持任何云端付费服务。
- **时间线**: MVP 目标 2026 Q3 前可用；首个现场测评窗口假设在 2026-09 前。
- **资源**: 小规模开发团队（1–2 名 Android 工程师 + 研究员兼 PO）；无专职 QA。
- **技术**:
  - 纯离线，无后端、无云同步；
  - 题目配置必须是**可人肉编辑**的 TXT 格式；
  - 使用 `MANAGE_EXTERNAL_STORAGE` 授权访问固定路径 `/sdcard/WenJuanPro/`；
  - 视觉时序抖动控制在 ±50ms 量级即可，不强制毫秒级精度。

### 关键假设
- 研究员可接受"直接写 TXT"的配题方式，并能按示例模板正确编写；
- 学生编号通过二维码发放，同一学号不会被多人同时使用；
- 现场可提供充电设备/电源，不需要考虑长时间电池续航极限；
- APK 仅通过手动侧载分发，不上架 Google Play 或国内主流应用市场（因此 `MANAGE_EXTERNAL_STORAGE` 权限无审核拒绝风险）；
- Jetpack Compose 原生动画 API 在目标机型上的视觉时序抖动足以满足 ±50ms 的研究可接受范围；
- Android SSAID 在目标设备生命周期内保持稳定，可作为结果文件命名的设备维度。

## 风险与待解问题

### 关键风险
- **`MANAGE_EXTERNAL_STORAGE` 授权流程摩擦**: 该权限需用户跳转系统设置手动授权，首次使用时研究员侧需做引导；部分 ROM（如某些定制 Android）对该权限策略更严格。**影响**: 首次部署需要研究员培训；需在 App 内提供清晰的授权引导页与失败回退文案。
- **TXT 格式错误导致现场失败**: 研究员手改 TXT 时可能破坏分隔符或字段顺序。**影响**: 开场即故障、数据作废；需强校验 + 清晰错误提示。
- **断点续答语义歧义**: "已记录题干阶段但未记录选项阶段"的半完成题如何处理？**影响**: 如果回放失误会破坏时序数据的可比性；当前按"整题重做"兜底，具体策略留待 PRD 细化。
- **SSAID 稳定性边缘情况**: 出厂重置、切换用户空间可能导致 SSAID 变更，从而结果文件设备维度失真。**影响**: 研究员需避免在测评周期内对设备执行恢复出厂。
- **视觉时序在极低端设备上的抖动**: 如果设备在高后台负载下出现 > 100ms 的卡顿，闪烁观感会劣化。**影响**: 建议在采购侧给出最低机型建议。
- **隐私/伦理风险**: 学生编号若映射到真实身份且 TXT 被泄露，存在合规风险。**影响**: 研究员端流程需约束。

### 待解问题
