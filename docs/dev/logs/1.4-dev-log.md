## Design Decisions (Gap Resolutions)

### [GAP-1] Duplicate-click de-dup strategy — **Resolution: one-shot via `lastSelectedConfigId`**

`ConfigListViewModel.cardClicked(configId)` short-circuits when `configId == lastSelectedConfigId`. This is
strictly one-shot: once the effect has been emitted, re-clicking the **same** valid card is a no-op. We
considered `throttleFirst` (time-window) and `distinctUntilChanged` (stream-based), but both still allow the
Effect channel to accumulate extras under racy UI events. One-shot is both deterministic and aligned with the
"student presses exactly once then scan page takes over" semantic. Selecting a different card still works
because the guard keys on the id, not a boolean.

### [GAP-2] N=100 performance sampling — **Plan: real-device, not CI**

Dev skeletons cover OOM avoidance on N=100 via `LazyColumn` (`key = sourceFileName`). Real-device sampling
(scan duration median + p95) must be captured on the research device once Android Studio can build locally.
Recording template: `{device, androidVersion, N, medianMs, p95Ms, result}`. Deferred (see §Deferred).

### [GAP-3] Hilt `@TestInstallIn` for `androidTest/` `DataModule` — **Resolution: avoid Hilt in tests**

Following Story 1.2's pattern (`PermissionScreenTest` drives `PermissionContent` directly), we test the
stateless `ConfigListContent` against prebuilt `ConfigListUiState` fixtures. This avoids the
`@HiltAndroidTest` + `@UninstallModules(DataModule::class)` + `TestDataModule` boilerplate, keeps instrumented
runs fast, and still exercises every UI branch the skeletons required. Navigation assertion uses
`TestNavHostController` wired to a minimal `NavHost` mirroring production routes.

## Notes on Blind-Spot / Edge Behaviors

- **BLIND-ERROR-002 (read-only disk)** — When `mkdirs()` returns false, `ensureConfigDir()` returns false.
  Downstream `loadAll()` still calls `listFiles` (returns `emptyList`) and VM renders `Empty`, not `Error`. This
  preserves the "graceful degradation" contract in Dev Notes §Unmet Precondition Handling.
- **BLIND-FLOW-002 (scope cancellation)** — `viewModelScope` cancellation during `refresh()` is safe: the
  launched job terminates cleanly, `finally { refreshInFlight = false }` still runs, and no post-cancel writes
  reach `_uiState`. Unit test reflects `ViewModel.clear()` to simulate.
- **BLIND-FLOW-003 (Back while sheet open)** — `BackHandler(enabled = sheetState != null)` emits
  `DismissSheet` and consumes the event, so the NavHost does not pop. No `BackHandler` is registered when no
  sheet is open, preserving default exit behavior on the first screen.
- **E2E-013 (retry after Timeout)** — Because `refreshInFlight` resets in `finally`, the retry button fires
  a fresh `ensureConfigDir` + `loadAll` pair. Verified via `coVerify(exactly = 2)`.
